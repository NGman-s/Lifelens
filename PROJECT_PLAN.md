# LifeLens (智眼生活) - 项目计划书

## 1. 项目概述 (Project Overview)

### 1.1 项目背景 (Background)
在快节奏的现代生活中，“吃得健康”已成为全民共识。然而，现有的饮食管理工具普遍存在**“高阻力”痛点**：
*   **记录繁琐**：用户需手动搜索食物、预估重量、输入热量，操作链路过长，难以坚持。
*   **反馈滞后**：往往在进食结束后才发现热量超标，缺乏事前的决策辅助。
*   **缺乏个性化**：通用的营养建议无法满足高血压、糖尿病或特定健身人群的差异化需求。

随着端侧计算能力的提升和多模态大模型（LMM）的成熟，利用 AI 技术重构饮食管理体验，实现从“手动记录”到“智能感知”的跨越，成为行业发展的必然趋势。

### 1.2 项目目的 (Purpose)
**LifeLens** 旨在打造一款基于多模态大模型的 **“Camera-First” (相机优先)** 智能饮食助手。
*   **极简交互**：通过“拍照即识别”替代繁琐录入，让健康记录无感化。
*   **即时决策**：在进食前提供可视化的营养分析（如“红绿灯”评级），辅助用户做出更健康的各种选择。
*   **端云协同**：结合端侧数据（运动消耗）与云端智能（营养分析），提供动态的健康闭环管理。

### 1.3 创新点 (Innovation Points)
1.  **交互模式创新 (Camera-First)**：
    *   打破传统 App 首页是菜单的定式，**启动即相机**。
    *   利用 AR 风格悬浮卡片（Glassmorphism），将营养数据直接叠加在实物画面上，实现“所见即所得”的沉浸式体验。
2.  **技术应用创新 (RAG-lite & Multimodal AI)**：
    *   引入 **Qwen-VL** 等先进视觉大模型，不仅能识别“是什么”（如牛肉面），更能理解“有多少”（份量预估）和“健康吗”（营养密度分析）。
    *   通过精心设计的 Prompt Engineering，迫使大模型输出结构化 JSON 数据，实现非结构化图像到结构化营养档案的实时转换。
3.  **架构设计创新 (Hybrid Architecture)**：
    *   采用 **Mock Mode (演示保障模式)**：内置基于手势触发的高保真演示模式，确保在大赛答辩现场零延迟、零失误，规避网络波动风险。

## 2. 技术方案 (Technical Solution)

### 2.1 总体架构 (Overall Architecture)
本项目采用前后端分离的现代化云端协同架构，重点适配 Android 平台：

*   **客户端 (Client)**：基于 **Uni-app (Vue 3 + Vite + Pinia)** 开发。
    *   负责图像采集、本地压缩算法（<100KB）、UI 渲染及本地数据存储。
    *   支持编译为 Android APK 及微信小程序。
*   **服务端 (Server)**：基于 **Python FastAPI** 构建的轻量级网关。
    *   部署于 Docker 容器中，负责请求转发、鉴权及与大模型 API 的交互。
*   **AI 引擎 (AI Engine)**：接入阿里云 **DashScope (Qwen3-VL-Flash)**。
    *   利用其强大的 OCR 和视觉推理能力，解析复杂菜品。

### 2.2 功能概述 (Function Overview)
1.  **AI 慧眼识餐**：核心功能。拍照 -> 云端分析 -> 返回热量/三大营养素/健康评分。
2.  **AR 营养透视**：在相机预览流上通过半透明卡片展示分析结果，视觉效果科幻、直观。
3.  **动态健康档案**：结合用户的身体数据（BMI、代谢率）提供个性化建议（如“今日碳水余额不足”）。
4.  **智能演示模式**：通过特定手势（点击 Logo 5次）进入 Mock Mode，调用本地预置的高质量数据集，用于评委演示。

### 2.3 关键技术 (Key Technologies)
1.  **端侧图像优化**：开发专用的 JS 图像压缩算法，在保留食物纹理细节的同时将图片体积压缩 90% 以上，确保移动网络下的秒级响应。
2.  **Prompt Engineering**：针对营养学场景定制 System Prompt，强制大模型输出标准 JSON 格式，包含 `calories`, `protein`, `fat`, `carbs`, `health_score` 等字段，极大降低了后处理难度。
3.  **Docker 容器化部署**：后端服务全量 Docker 化，确保开发、测试、生产环境的一致性，便于快速扩容。

### 2.4 OPPO 生态集成规划 (Integration Plan)
为了契合大赛对 OPPO ColorOS 能力的要求，项目规划以下集成：
1.  **OPPO Health Service**：调用健康服务 SDK 获取用户当日的 **实时步数** 和 **活动能量消耗**。
    *   *应用场景*：动态调整用户的“今日推荐摄入量”，实现“摄入-消耗”的动态平衡。
2.  **ColorOS 端侧 AI**：利用手机本地 NPU 能力处理基础 OCR（如识别包装袋文字）或条形码扫描。
    *   *优势*：减少云端 API 调用成本，提升隐私安全性。

## 3. 项目计划 (Project Plan)

*   **技术可行性**：

## 4. 商业价值与社会效益 (Commercial & Social Value)

### 4.1 商业模式 (Business Model)
本项目采用 **“Freemium” (免费+增值服务)** 模式：
*   **基础版 (免费)**：包含无限次 AI 拍照识别、基础营养记录。快速获取海量用户，积累数据。
*   **专业版 (订阅制)**：提供周/月度深度营养分析报告、微量元素（维生素/矿物质）分析、导出 PDF 报告给医生/教练。
*   **流量变现 (规划)**：基于用户饮食偏好的健康食品（低脂零食、代餐）精准推荐。

### 4.2 市场前景 (Market Analysis)
*   **健康意识觉醒**：随着肥胖率上升，减脂人群基数庞大。
*   **老龄化社会需求**：糖尿病、高血压患者需要严格的饮食监控，LifeLens 可作为其贴身“数字营养师”。

### 4.3 社会效益 (Social Impact)
*   **降低医疗负担**：通过预防性的日常饮食管理，降低慢性病发病率。
*   **提升健康素养**：以寓教于乐的方式（拍照即学）普及营养学知识，帮助公众建立科学的饮食观。
